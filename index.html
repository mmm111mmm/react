Below are versions of this app.

Each version of the app covers a new topic from this week.

Eventually I will cover all the topics.

    live site

    Explanation:
    1. We create a Dog component. And use two of them.

    3. Press 'feed' on a dog, and the onClick event is called.
    4. When we then update the 'hungry' state.
  • Dogapp v2 - lists and conditional rendering
    live site
    source code on repl.it

    Explanation:
    1. We create a new Playtime component.
    2. This has three 'treat' buttons: stroke, play ball, give oranage.
    3. When a button is pressed, the treat is added to a list.
    4. If the last item of the list is 'give orange', we render "ORANGES!!!".
  • Dogapp v3 - forms
    live site

    Explanation:
    1. We change the Playtime component.
    2. We no longer have buttons in the Playtime component.
    3. We have a html select
    4. And we have a html input

  • Dogapp v4 - simple routing
    live site
    source code on repl.it

    Explanation:
    1. We add an AboutScreen component.
    2. We add an DogsScreen component - this has Kira and Tez in.
    3. Once we click on the 'about' link, we go to a new url (../about), and therefore the AboutScreen component
    4. Once we click on the 'dogs' link, we go to a new url (../dogs), and therefore the DogsScreen component
  • Dogapp v5 - routing with url parameters
    live site
    source code on repl.it

    Explanation:
    1. Once we click on the 'dogs' link, we go to a new url (../dogs/pink), and therefore the DogsScreen component will have a pink background
  • Dogapp v6 - lifting up state
    live site
    source code on repl.it

    Explanation:
    1. The Dog component now has no state.
    2. All the state is held in DogsScreen.
    3. DogsScreen passes the state to the Dog components as props.
    4. When the Dog component wants to update the state, it calls a method on DogsScreen.
